# 文件消息与文本组合功能

## 功能说明

现在支持在同一条消息中同时发送文件和文本说明，用户可以为文件添加标题或描述。

## 使用方法

### 1. 上传文件并添加说明

1. 点击 📎 按钮选择文件
2. 文件上传后会显示在输入框上方
3. 在输入框中输入文字说明（可选）
4. 点击 Send 发送

### 2. 界面展示

```
┌─────────────────────────────────────┐
│ 📎 document.pdf              ✕     │  ← 文件预览
├─────────────────────────────────────┤
│ [📎] [😀] [这是一个重要文档...] [Send] │  ← 输入框
└─────────────────────────────────────┘
```

### 3. 消息显示

#### 带说明的图片
```
┌─────────────────────┐
│   [图片预览]         │
│                     │
│ 这是我的新照片       │  ← 用户输入的说明
│ photo.jpg (2.5 MB)  │  ← 文件信息
└─────────────────────┘
```

#### 带说明的文件
```
┌─────────────────────┐
│ 📎 document.pdf  ⬇️ │
│                     │
│ 这是项目文档         │  ← 用户输入的说明
└─────────────────────┘
```

#### 不带说明的文件
```
┌─────────────────────┐
│ 📎 document.pdf  ⬇️ │
│ document.pdf (1 MB) │  ← 只显示文件名
└─────────────────────┘
```

## 功能特性

### 文件附件预览
- ✅ 显示文件图标（根据类型）
  - 🖼️ 图片
  - 🎵 音频
  - 🎬 视频
  - 📎 其他文件
- ✅ 显示文件名
- ✅ 移除按钮（✕）

### 输入框增强
- ✅ 附件时显示不同的占位符
  - 无附件: "Type a message..."
  - 有附件: "Add a caption (optional)..."
- ✅ 附件后禁用上传按钮（一次只能附加一个文件）
- ✅ 可以只发送文件（不输入文字）
- ✅ 可以发送文件+文字说明

### 消息显示
- ✅ 文件预览（图片/音频/视频）
- ✅ 文字说明显示在文件下方
- ✅ 分隔线区分文件和说明
- ✅ 如果说明就是文件名，则不重复显示

## 技术实现

### 状态管理

```typescript
const [attachedFile, setAttachedFile] = useState<{
  file: File
  fileInfo: {
    file_id: string
    filename: string
    content_type: string
    file_size: number
    url: string
  }
} | null>(null)
```

### 发送逻辑

```typescript
// 1. 上传文件（不立即发送）
const fileInfo = await apiClient.uploadFile(file)
setAttachedFile({ file, fileInfo })

// 2. 用户添加说明（可选）
// 用户在输入框输入文字

// 3. 点击 Send 发送
const body = messageText.trim() || fileInfo.filename
const content = {
  msgtype: 'm.image', // 或 m.audio, m.video, m.file
  body: body,         // 文字说明或文件名
  url: fileInfo.url,
  info: { size, mimetype }
}
```

### 消息结构

```json

  "info": {
    "size": 2621440,
    "mimetype": "image/jpeg"
  }

```

## 使用场景

### 场景 1: 分享照片
```
用户: 上传照片 → 输入 "今天的晚餐" → 发送
显示: [照片] + "今天的晚餐"
```

### 场景 2: 发送文档
```
用户: 上传 PDF → 输入 "请查看这个报告" → 发送
显示: [PDF 文件] + "请查看这个报告"
```

### 场景 3: 快速分享
```
用户: 上传文件 → 直接发送（不输入文字）
显示: [文件] + 文件名
```

### 场景 4: 分享音乐
```
用户: 上传 MP3 → 输入 "推荐这首歌" → 发送
显示: [音频播放器] + "推荐这首歌"
```

## 注意事项

1. **一次一个文件**: 当前只支持一次附加一个文件
2. **文件大小**: 仍然限制在 50MB
3. **说明长度**: 建议不超过 500 字符
4. **文件名显示**: 如果说明就是文件名，不会重复显示

## 对比

### 之前的方式
```

2. 无法添加说明

```

```
1. 点击 📎 → 选择文件 → 显示预览
2. 输入说明（可选）
3. 点击 Send → 文件和文字一起发送
```

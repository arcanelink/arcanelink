# 文件发送功能快速测试指南

## 启动服务

```bash
# 1. 启动后端服务
docker-compose up -d

# 2. 启动 Web 客户端
cd web-client
npm install
npm run dev
```

## 测试步骤

### 1. 注册/登录

1. 打开浏览器访问 http://localhost:3000
2. 注册两个测试账号：
   - 用户 A: testuser1
   - 用户 B: testuser2

### 2. 测试私聊文件发送

1. 用户 A 登录
2. 点击 "New Chat" 创建与 testuser2 的私聊
3. 点击输入框左侧的 📎 按钮
4. 选择一个文件（建议先测试小文件，如图片）
5. 等待上传完成
6. 文件消息自动发送

### 3. 测试不同文件类型

#### 测试图片
1. 上传 .jpg, .png, .gif 等图片文件
2. 验证图片是否正确显示预览
3. 验证文件名和大小是否显示

#### 测试音频
1. 上传 .mp3, .wav 等音频文件
2. 验证音频播放器是否显示
3. 点击播放按钮测试播放

#### 测试视频
1. 上传 .mp4, .webm 等视频文件
2. 验证视频播放器是否显示
3. 点击播放按钮测试播放

#### 测试通用文件
1. 上传 .pdf, .doc, .zip 等文件
2. 验证文件卡片是否显示
3. 点击下载按钮测试下载

### 4. 测试群聊文件发送

1. 创建一个房间
2. 邀请 testuser2 加入
3. 在房间中发送文件
4. 用 testuser2 登录验证是否能看到文件

### 5. 测试边界情况

#### 大文件测试
1. 尝试上传 50MB 的文件（应该成功）
2. 尝试上传 51MB 的文件（应该提示错误）

#### 网络中断测试
1. 开始上传大文件
2. 断开网络
3. 验证错误提示

#### 多文件测试
1. 连续发送多个文件
2. 验证所有文件都正确显示

### 6. 验证功能

#### 发送方验证
- [ ] 文件上传成功
- [ ] 文件消息正确显示
- [ ] 文件信息（名称、大小）正确
- [ ] 可以下载自己发送的文件

#### 接收方验证
- [ ] 能收到文件消息
- [ ] 文件正确显示
- [ ] 可以下载文件
- [ ] 图片/音频/视频可以预览/播放

#### UI/UX 验证
- [ ] 上传按钮位置合理
- [ ] 上传中有加载提示（⏳）
- [ ] 文件消息样式美观
- [ ] 悬停效果正常
- [ ] 移动端适配良好

## 常见问题排查

### 文件上传失败

1. 检查后端服务是否运行：
```bash
docker-compose ps
```

2. 检查 API Gateway 日志：
```bash
docker-compose logs api-gateway
```

3. 检查文件存储目录权限：
```bash
ls -la data/files
```

### 文件无法下载

1. 检查文件是否存在：
```bash
docker exec -it mbot-api-gateway ls -la /data/files
```

2. 检查数据库中的文件记录：
```bash
docker exec -it mbot-postgres psql -U mbot -d mbot -c "SELECT * FROM file_storage;"
```

### 图片/视频无法显示

1. 检查浏览器控制台是否有错误
2. 验证文件 URL 是否正确
3. 检查 CORS 配置

## 性能测试

### 上传速度测试
1. 上传 1MB 文件，记录时间
2. 上传 10MB 文件，记录时间
3. 上传 50MB 文件，记录时间

### 并发测试
1. 同时打开多个浏览器标签
2. 同时上传文件
3. 验证是否有冲突

### 存储测试
1. 上传相同文件多次
2. 验证是否去重（检查存储目录）
3. 验证数据库记录

## 自动化测试脚本

使用后端测试脚本：

```bash
chmod +x scripts/test_file_upload.sh
./scripts/test_file_upload.sh
```

该脚本会自动测试：
- 用户注册
- 文件上传
- 文件下载
- 私聊发送文件
- 群聊发送文件

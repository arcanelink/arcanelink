## 协议简介

本协议是一个基于Matrix协议改进的分布式即时通讯协议，旨在简化Matrix的复杂性，同时保持其分布式架构的优势。

### 设计目标

1. **分布式架构**：保持服务器联邦模型，任何人都可以运行自己的服务器
2. **简化设计**：移除不必要的复杂性，提升性能和易用性
3. **双通道模型**：私聊和群聊使用不同的处理机制
4. **易于实现**：降低实现难度，使用标准HTTP协议

### 核心特性

- **私聊不使用Room**：私聊消息直接点对点路由，无需创建Room对象
- **群聊使用Room**：群聊保留Room概念，维护成员列表和历史记录
- **HTTP长轮询**：使用HTTP长轮询替代WebSocket，更简单、更兼容
- **可选加密**：不强制要求端到端加密，降低实现复杂度
- **服务器联邦**：支持跨服务器通信，用户可以与任何服务器的用户交流

## 与Matrix的主要区别

| 特性 | Matrix | 本协议 |
|------|--------|--------|
| 私聊机制 | 使用Room | 直接点对点路由 |
| 群聊机制 | 使用Room | 使用Room |
| 通信方式 | WebSocket/HTTP | HTTP长轮询 |
| 加密要求 | 支持E2EE | 可选，不强制 |
| 消息模型 | 统一事件模型 | 双通道模型 |

## 基本概念

### 用户标识

用户ID格式：`@username:domain.com`

示例：`@alice:example.com`

### 服务器标识

服务器ID即为域名：`domain.com`

### Room标识

Room ID格式：`!roomid:domain.com`

示例：`!abc123:example.com`

## 协议层次

```
客户端应用层
    ↓
客户端API层 (HTTP长轮询 + REST API)
    ↓
Homeserver (用户管理、消息路由、存储)
    ↓
联邦协议层 (服务器间通信)
    ↓
传输层 (HTTP/1.1 或 HTTP/2)
```

## 消息流向

### 私聊消息流
```
发送方客户端 → 发送方HS → 接收方HS → 接收方客户端
```

### 群聊消息流
```
发送方客户端 → Room所在HS → 各成员的HS → 各成员客户端
```
